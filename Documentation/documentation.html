    Database

    From the Database sevice of Firebase we chose to use Realtime Database for storing user's sites' data. Realtime Database uses a noSql type of database wich greatly applies to our type of application - we need to store data under each user's unique UID (the data stored here is not only descriptional, so it's not only data that the user needs but also data that we need such as their private and public keys for encrypting and decrypting their data) and also the public and private keys of the server used for crypting and decrypting data sent over the network:

User Interaction

The user will create an account on our site. At that moment, an alghoritm will link his data to a RSA set of keys. When logged in, the user can request a javascript chunk of code that he can insert in his form-page so that the sent data will be cripted with the AES alghoritm, using the RSA key. The user can also view the data that had passed his form and was successsfully encripted and stored in the data base.
