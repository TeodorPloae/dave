    On our first encounter with the project we tryed to determine what is the concept of what whe are implementeing. We soon realised that we need an tackle plan for this. Thus we started developing concepts and ideas of how the project should look and behave.

    We started simple! The fact that a form was needed was obviously. A core concept started to matearialize. It was ugly and simple but a starting point none the less. We tryed to make it accesible for every type of user by assigning each input with an appropriate label.

    UPDATE: Added some good old javascript for the initial form so that you cand switch between Login and Sign Up.

User Interaction

The user will create an account on our site. At that moment, an alghoritm will link his data to a RSA set of keys. When logged in, the user can request a javascript chunk of code that he can insert in his form-page so that the sent data will be cripted with the AES alghoritm, using the RSA key. The user can also view the data that had passed his form and was successsfully encripted and stored in the data base.
